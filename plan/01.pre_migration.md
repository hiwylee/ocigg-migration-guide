```sql
FROM DBA_SEGMENTS;

-- 사용자 스키마별 용량
SELECT OWNER,
       ROUND(SUM(BYTES)/1024/1024/1024, 2) AS GB,

)
```

### 1-4. 특수 오브젝트 현황 파악

```sql
-- [소스] PK 없는 테이블 목록 (Supplemental Log 추가 대상)
SELECT T.OWNER, T.TABLE_NAME
FROM DBA_TABLES T
WHERE T.OWNER NOT IN (
    'SYS','SYSTEM','OUTLN','DBSNMP','APPQOSSYS','AUDSYS','CTXSYS',
    'DBSFWUSER','DVSYS','EXFSYS','GGSYS','GSMADMIN_INTERNAL','LBACSYS',
    'MDSYS','OJVMSYS','OLAPSYS','ORDDATA','ORDSYS','WMSYS','XDB','XS$NULL',
    'GGADMIN','GGS_TEMP','MIGRATION_USER'
)
AND T.TEMPORARY = 'N'
AND T.EXTERNAL = 'NO'
AND NOT EXISTS (
    SELECT 1 FROM DBA_CONSTRAINTS C
    WHERE C.OWNER = T.OWNER AND C.TABLE_NAME = T.TABLE_NAME
    AND C.CONSTRAINT_TYPE = 'P'
)
ORDER BY T.OWNER, T.TABLE_NAME;
```

| 항목 | 건수 | 비고 |
|------|-----|------|
| PK 없는 테이블 | | ALL COLUMNS Supplemental Log 필요 |

```sql
    'MDSYS','OJVMSYS','OLAPSYS','ORDDATA','ORDSYS','WMSYS','XDB'
```

### 5-1. 소스 RDS — GGADMIN 계정 생성

```sql
-- [소스] GGADMIN 계정 생성 및 권한 부여
CREATE USER GGADMIN IDENTIFIED BY "<strong_password>"
    DEFAULT TABLESPACE USERS
    TEMPORARY TABLESPACE TEMP
    PROFILE DEFAULT;

GRANT CREATE SESSION       TO GGADMIN;
GRANT ALTER SESSION        TO GGADMIN;
GRANT SELECT ANY DICTIONARY TO GGADMIN;
GRANT SELECT ANY TABLE     TO GGADMIN;
GRANT FLASHBACK ANY TABLE  TO GGADMIN;
GRANT EXECUTE ON DBMS_FLASHBACK TO GGADMIN;
GRANT SELECT ON V_$DATABASE         TO GGADMIN;
GRANT SELECT ON V_$LOG              TO GGADMIN;
GRANT SELECT ON V_$LOGFILE          TO GGADMIN;
GRANT SELECT ON V_$ARCHIVED_LOG     TO GGADMIN;
```

## STEP 11. Streams Pool 메모리 설정

**목적**: Oracle SE는 GG용 Streams Pool을 수동으로 설정해야 함 (자동 할당 안 됨)
**담당**: 소스 DBA
**실행 환경**: `[소스]` + `[AWS콘솔]`
**⚠️ SE 제약**: SE에서 Streams Pool은 자동 SGA 관리 대상 아님 — 반드시 수동 설정

```sql
-- [소스] 현재 streams_pool_size 확인
SELECT NAME, VALUE/1024/1024 AS MB
FROM V$PARAMETER WHERE NAME = 'streams_pool_size';
```

**현재값**: ____________ MB

### 11-1. 필요 크기 산정

```
STREAMS_POOL_SIZE ≈ (Redo rate MB/s × 60 × 5) × 1.5
                   = Redo 5분치 × 안전계수 1.5
```

```sql
-- [소스] Redo 생성 속도 확인 (최근 1시간 평균, MB/s)
SELECT ROUND(SUM(BLOCKS * BLOCK_SIZE) / 3600 / 1024 / 1024, 2) AS REDO_RATE_MB_PER_SEC
FROM V$ARCHIVED_LOG
WHERE FIRST_TIME >= SYSDATE - 1/24
  AND STANDBY_DEST = 'NO';
```

**Redo rate**: ____________ MB/s → STREAMS_POOL_SIZE 산정값: ____________ MB

설정이 필요한 경우:

```
[AWS콘솔]
1. RDS → 파라미터 그룹 → 해당 DB의 파라미터 그룹 선택
2. streams_pool_size 검색 → 값 입력
   - 최소: 268435456 (256MB)
   - 권장: 산정값 이상 (최소 512MB)
   - 예시: Redo rate 2 MB/s → 2×60×5×1.5 = 900MB → 1GB(1073741824) 설정
3. 변경 적용 → DB 재기동 (Pending Reboot 발생)
```

> **⚠️ AWS RDS 주의**: `ALTER SYSTEM SET streams_pool_size` 직접 실행 불가 — 반드시 Parameter Group 사용

```sql
-- [소스] 재기동 후 확인
SELECT NAME, VALUE/1024/1024 AS MB
FROM V$PARAMETER WHERE NAME = 'streams_pool_size';
```

| 항목 | 기대값 | 실제값 | 판정 |
